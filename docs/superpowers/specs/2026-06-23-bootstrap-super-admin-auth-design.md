# Bootstrap Super Admin Authentication

## Goal

Use Google only to verify identity. The active term roster decides who may sign in as a member. A single configured email bootstraps the first super admin so the roster can be managed through the UI.

## Configuration

Add `AUTH_BOOTSTRAP_SUPER_ADMIN_EMAIL`.

Remove `AUTH_ALLOWLIST_EMAILS`.

Keep `AUTH_ALLOWED_DOMAINS` optional. When empty, it adds no domain restriction. When set, the verified Google email must use one of the listed domains unless it is the bootstrap super admin.

## Sign-In Rules

1. Require the Google provider, a verified email, and a normalized email address.
2. Allow the configured bootstrap email even when it is absent from the active term roster.
3. For every other email, apply the optional domain restriction and require an active term roster entry.
4. Existing super admins retain the current roster bypass.

## Bootstrap Role

When Auth.js creates the member matching `AUTH_BOOTSTRAP_SUPER_ADMIN_EMAIL`, assign the existing `super` role. The operation must be safe to repeat and must not create a duplicate role assignment.

The bootstrap configuration grants access only to the exact normalized email address.

## Local Environment

`.env.local` will contain:

- A generated `AUTH_SECRET`.
- `AUTH_BOOTSTRAP_SUPER_ADMIN_EMAIL=charles.joshua.uy@student.ateneo.edu`.
- Localhost values for `AUTH_URL` and `APP_BASE_URL`.
- Empty slots for Google client ID, Google client secret, and the shared API token.
- Shared mode pointed at `https://beta.ateneocode.org`.

The Google credentials and shared API token cannot be generated by the application.

## Verification

Focused tests will verify:

- Empty allowed domains do not reject a verified Google identity.
- Configured domains still restrict ordinary users.
- The bootstrap email bypasses the roster.
- The bootstrap member receives the `super` role without duplicate assignment.
- Ordinary users still require the active term roster.

Run the focused tests, `pnpm lint`, and `pnpm build`. Check graph consistency with `graphify update .`.
